# Security Policy

OrbMitra is an open-source project, so we take security issues seriously.

## Reporting a Vulnerability

If you believe you found a security issue, do not open a public issue with exploit details.

Please report it privately through GitHub Security Advisories if available, or through the maintainer's private contact channel.

Include as much detail as you can:

- what the issue is
- the affected file or route
- steps to reproduce
- the potential impact
- any safe proof-of-concept output

## What Counts as Security-Sensitive

Examples include:

- authentication and session handling
- exposed secrets or credentials
- XSS or injection flaws
- unauthorized admin access
- unsafe file or database access
- dependency vulnerabilities that can be exploited

## Safe Disclosure

Please give the maintainers a reasonable chance to fix the issue before posting technical details publicly.
